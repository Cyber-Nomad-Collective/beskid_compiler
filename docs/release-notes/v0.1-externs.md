# v0.1 externs: release notes

Highlights:
- Language-level extern calls: `C.getpid()` lowers to a direct external call when `C` is an extern contract
- Frontend validation of extern contracts (Abi, Library, FFI types)
- Linux-only dynamic linking via `--features extern_dlopen`
- Security allow/deny patterns via environment variables

Whats included
- Engine support: dynamic symbol resolution, per-artifact JIT module with pre-registered externs
- Analysis: diagnostics for invalid extern contracts (e.g., wrong ABI, missing library, disallowed types)
- Tests: endtoend `getpid` call, resolution-only flows, policy enforcement tests
- Docs and examples: policy, cookbook, examples/extern

Limitations
- Linuxonly in this release
- No varargs support
- Raw pointers supported as `ref u8` for params only
- No permethod link name override (future work)

Revert strategy
- Disable extern tests by omitting `--features extern_dlopen`
- Security: set `BESKID_EXTERN_ALLOW`/`DENY` to lock down symbols

Next up
- Windows/macOS backends
- Varargs and `LinkName` permethod
- Safer pointer helpers for byte slices to enable `libc.write` and friends
